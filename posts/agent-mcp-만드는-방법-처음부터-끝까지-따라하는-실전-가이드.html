Agent MCP 만드는 방법: 처음부터 끝까지 따라하는 실전 가이드

AI 에이전트를 더 강력하게 만들어주는 MCP(Model Context Protocol) 서버를 직접 구축하는 방법을 단계별로 알아봅니다. 기본 개념부터 실제 코드 작성, 클라이언트 연동까지 실무에 바로 적용할 수 있는 내용을 정리했습니다.

MCP란 무엇인가?

MCP(Model Context Protocol)는 2024년 11월 Anthropic이 공개한 오픈 표준 프로토콜로, AI 에이전트(LLM)가 외부 데이터 소스나 도구에 안전하고 표준화된 방식으로 접근할 수 있도록 설계되었습니다. 쉽게 말해 "AI를 위한 USB-C 포트" 라고 비유할 수 있습니다.

기존에는 각 AI 애플리케이션마다 데이터 소스나 API 연동을 위한 커스텀 코드를 별도로 작성해야 했지만, MCP를 사용하면 한 번 만든 서버를 Claude Desktop, Cursor, Continue 등 여러 클라이언트에서 동일하게 활용할 수 있습니다.

MCP의 핵심 구성 요소

MCP는 클라이언트-서버 아키텍처로 동작하며, 다음 세 가지 주요 개념을 제공합니다.

  • Tools(도구): LLM이 호출할 수 있는 함수 (예: 날씨 조회, DB 쿼리)
  • Resources(리소스): LLM이 읽을 수 있는 데이터 (예: 파일, API 응답)
  • Prompts(프롬프트): 재사용 가능한 프롬프트 템플릿

MCP 서버를 만들기 전 준비사항

MCP 서버는 공식적으로 Python과 TypeScript SDK를 지원합니다. 이 글에서는 가장 빠르게 시작할 수 있는 Python 기반 예제로 진행하겠습니다.

필수 환경

  • Python 3.10 이상
  • uv 패키지 매니저 (Anthropic이 권장)
  • Claude Desktop 또는 MCP 호환 클라이언트
# uv 설치 (macOS/Linux)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 프로젝트 초기화
uv init my-mcp-server
cd my-mcp-server
uv add "mcp[cli]" httpx

1단계: 기본 MCP 서버 구조 만들기

FastMCP는 공식 SDK가 제공하는 고수준 API로, 데코레이터 기반으로 손쉽게 서버를 만들 수 있습니다.

# server.py
from mcp.server.fastmcp import FastMCP

# MCP 서버 인스턴스 생성
mcp = FastMCP("my-agent-server")

if __name__ == "__main__":
    # stdio 방식으로 서버 실행
    mcp.run(transport="stdio")

이 단 5줄의 코드만으로도 유효한 MCP 서버가 됩니다. 다만 아직 어떠한 도구도 등록되지 않았으므로, 실제로 활용하려면 기능을 추가해야 합니다.

2단계: Tool(도구) 추가하기

LLM이 호출할 수 있는 함수를 정의해보겠습니다. 예시로 날씨 조회 도구를 만들어 보겠습니다.

import httpx
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("weather-server")

@mcp.tool()
async def get_weather(city: str) -> str:
    """특정 도시의 현재 날씨를 조회합니다.

    Args:
        city: 날씨를 조회할 도시명 (영문)
    """
    url = f"https://wttr.in/{city}?format=3"
    async with httpx.AsyncClient() as client:
        response = await client.get(url)
        return response.text

if __name__ == "__main__":
    mcp.run(transport="stdio")

중요한 포인트

  • Docstring이 곧 LLM에게 전달되는 설명입니다. 도구의 용도와 인자를 명확히 작성해야 LLM이 적절히 사용할 수 있습니다.
  • 타입 힌트는 필수입니다. city: str처럼 명시해야 MCP가 자동으로 JSON 스키마를 생성합니다.
  • 함수는 동기/비동기 모두 지원되지만, I/O가 발생하는 경우 async를 권장합니다.

3단계: Resource(리소스) 추가하기

리소스는 LLM이 "읽기 전용"으로 참조할 수 있는 데이터입니다. 파일, DB 레코드, 설정값 등을 노출할 때 사용합니다.

@mcp.resource("config://app")
def get_app_config() -> str:
    """애플리케이션 설정 값을 반환합니다."""
    return "버전: 1.0.0, 환경: production"

@mcp.resource("user://{user_id}/profile")
def get_user_profile(user_id: str) -> str:
    """사용자 프로필을 동적으로 반환합니다."""
    return f"사용자 {user_id}의 프로필 정보"

URI 패턴에 {user_id} 같은 템플릿 변수를 사용하면 동적 리소스도 만들 수 있습니다.

4단계: Prompt(프롬프트) 추가하기

자주 사용하는 프롬프트 패턴을 템플릿으로 등록할 수 있습니다.

@mcp.prompt()
def code_review(code: str) -> str:
    """코드 리뷰를 위한 프롬프트를 생성합니다."""
    return f"""다음 코드를 리뷰해주세요. 
보안, 성능, 가독성 관점에서 개선점을 알려주세요.

{code}

"""

5단계: Claude Desktop에 연동하기

작성한 MCP 서버를 Claude Desktop에서 사용하려면 설정 파일을 수정해야 합니다.

macOS 경로: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows 경로: %APPDATA%\Claude\claude_desktop_config.json

{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": [
        "--directory",
        "/절대/경로/my-mcp-server",
        "run",
        "server.py"
      ]
    }
  }
}

설정 후 Claude Desktop을 완전히 재시작하면 도구 아이콘이 활성화되며, 채팅창에서 "서울 날씨 알려줘"라고 입력하면 LLM이 자동으로 get_weather 도구를 호출합니다.

6단계: 디버깅과 테스트

MCP 서버 개발 시 가장 강력한 도구는 공식 MCP Inspector입니다.

npx @modelcontextprotocol/inspector uv run server.py

이 명령을 실행하면 웹 UI가 뜨고, 등록된 도구·리소스·프롬프트를 직접 호출하면서 응답을 검증할 수 있습니다. 실제 LLM에 연결하기 전에 이 단계에서 충분히 테스트하는 것이 권장됩니다.

실전 팁: 좋은 MCP 서버를 만드는 원칙

수십 개의 공개된 MCP 서버를 분석해보면 다음과 같은 공통점이 발견됩니다.

  1. 도구 이름은 명확하게: get_user보다 fetch_user_by_email이 더 좋습니다.
  2. 에러 처리는 사람이 읽기 쉬운 메시지로: LLM은 에러 메시지를 보고 다음 행동을 결정하기 때문입니다.
  3. 반환 데이터는 LLM 친화적으로: 거대한 JSON보다는 핵심 정보만 추려 반환하는 것이 토큰 효율적입니다.
  4. 민감한 작업에는 confirmation 패턴 적용: 파일 삭제, DB 변경 같은 작업은 별도 검증 단계를 둡니다.
  5. 단일 책임 원칙: 하나의 서버에 너무 많은 도구를 넣지 말고 도메인별로 분리합니다.

마무리: 어떤 MCP 서버를 만들까?

MCP의 진짜 매력은 재사용성에 있습니다. 본인의 업무에서 반복되는 작업이 있다면 그것이 곧 좋은 MCP 서버 후보입니다. 예를 들어:

  • 사내 위키 검색 MCP
  • Jira/Notion 연동 MCP
  • 로그 분석 및 알림 MCP
  • 개인 메모/할일 관리 MCP

공식 GitHub 저장소(modelcontextprotocol/servers)에는 이미 수십 개의 레퍼런스 구현이 공개되어 있으니, 직접 만들기 전에 참고하면 시행착오를 크게 줄일 수 있습니다.

MCP는 아직 빠르게 진화하고 있는 표준이지만, 한 번 익혀두면 AI 에이전트 시대의 핵심 인프라 기술로서 큰 경쟁력이 됩니다. 오늘 소개한 5단계 흐름을 따라 첫 번째 서버를 만들어보세요. 의외로 한 시간이면 충분합니다.
