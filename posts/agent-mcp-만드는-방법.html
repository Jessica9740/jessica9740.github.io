AI 에이전트를 위한 MCP 서버 만드는 방법: 처음부터 끝까지 완벽 가이드

Anthropic이 공개한 MCP(Model Context Protocol)는 AI 에이전트가 외부 도구와 데이터를 표준화된 방식으로 사용할 수 있게 해주는 프로토콜입니다. 이 글에서는 MCP의 핵심 개념부터 직접 서버를 구현하고 Claude Desktop에 연결하는 실전 단계까지 차근차근 알아봅니다.

MCP란 무엇인가?

MCP(Model Context Protocol)는 2024년 11월 Anthropic이 오픈소스로 공개한 개방형 표준 프로토콜입니다. 한마디로 표현하면 "AI 모델과 외부 데이터 소스를 연결하는 USB-C"라고 할 수 있습니다.

기존에는 AI 에이전트가 데이터베이스, 파일 시스템, API 같은 외부 리소스에 접근하려면 각각의 통합을 직접 구현해야 했습니다. MCP는 이 과정을 표준화하여, 한 번 만든 MCP 서버를 Claude Desktop, Cursor, Continue 등 MCP를 지원하는 모든 클라이언트에서 재사용할 수 있도록 합니다.

MCP의 핵심 구성 요소

MCP는 세 가지 주요 컴포넌트로 구성됩니다.

  • Host: AI 모델을 실행하는 애플리케이션 (예: Claude Desktop)
  • Client: Host 내부에서 서버와 통신하는 컴포넌트
  • Server: 도구(Tools), 리소스(Resources), 프롬프트(Prompts)를 제공하는 프로그램

서버가 제공하는 세 가지 기본 기능은 다음과 같습니다.

  1. Tools: 모델이 호출할 수 있는 함수 (예: 날씨 조회, 이메일 전송)
  2. Resources: 모델이 읽을 수 있는 데이터 (예: 파일, DB 레코드)
  3. Prompts: 재사용 가능한 프롬프트 템플릿

개발 환경 준비하기

MCP 서버는 공식적으로 Python과 TypeScript SDK를 지원합니다. 이번 가이드에서는 Python을 사용하겠습니다.

필수 도구 설치

# uv 패키지 매니저 설치 (Anthropic 공식 권장)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 프로젝트 생성
uv init my-mcp-server
cd my-mcp-server

# MCP SDK 설치
uv add "mcp[cli]" httpx

uv는 Python 패키지 매니저로, 의존성 관리와 가상 환경 구성을 매우 빠르게 처리합니다. Anthropic의 공식 문서에서도 권장하는 도구입니다.

첫 번째 MCP 서버 구현하기

이제 실제로 동작하는 간단한 MCP 서버를 만들어 봅시다. 예제로는 공개 API를 활용한 날씨 정보 조회 도구를 구현하겠습니다.

기본 서버 구조

server.py 파일을 생성하고 다음과 같이 작성합니다.

from mcp.server.fastmcp import FastMCP
import httpx

# 서버 인스턴스 생성
mcp = FastMCP("weather-server")

# 상수 정의
NWS_API_BASE = "https://api.weather.gov"
USER_AGENT = "weather-app/1.0"

async def make_request(url: str) -> dict | None:
    """NWS API에 요청을 보내고 결과를 반환"""
    headers = {
        "User-Agent": USER_AGENT,
        "Accept": "application/geo+json"
    }
    async with httpx.AsyncClient() as client:
        try:
            response = await client.get(url, headers=headers, timeout=30.0)
            response.raise_for_status()
            return response.json()
        except Exception:
            return None

Tool 정의하기

@mcp.tool() 데코레이터로 AI가 호출할 수 있는 도구를 정의합니다.

@mcp.tool()
async def get_forecast(latitude: float, longitude: float) -> str:
    """특정 위치의 날씨 예보를 가져옵니다.

    Args:
        latitude: 위도 (예: 37.5665)
        longitude: 경도 (예: 126.9780)
    """
    # 1단계: 해당 위치의 예보 URL 가져오기
    points_url = f"{NWS_API_BASE}/points/{latitude},{longitude}"
    points_data = await make_request(points_url)

    if not points_data:
        return "위치 정보를 가져올 수 없습니다."

    # 2단계: 실제 예보 가져오기
    forecast_url = points_data["properties"]["forecast"]
    forecast_data = await make_request(forecast_url)

    if not forecast_data:
        return "예보 정보를 가져올 수 없습니다."

    # 결과 포맷팅
    periods = forecast_data["properties"]["periods"][:5]
    forecasts = []
    for period in periods:
        forecast = f"""
{period['name']}:
온도: {period['temperature']}°{period['temperatureUnit']}
바람: {period['windSpeed']} {period['windDirection']}
예보: {period['detailedForecast']}
        """
        forecasts.append(forecast)

    return "\n---\n".join(forecasts)


if __name__ == "__main__":
    mcp.run(transport='stdio')

여기서 중요한 점은 함수의 docstring입니다. AI 모델은 이 설명을 읽고 도구를 언제, 어떻게 사용할지 결정하기 때문에 명확하고 구체적으로 작성해야 합니다.

Resource와 Prompt 추가하기

도구 외에 리소스와 프롬프트도 정의할 수 있습니다.

@mcp.resource("config://app")
def get_config() -> str:
    """애플리케이션 설정 정보를 반환"""
    return "App version: 1.0.0\nMode: production"

@mcp.prompt()
def review_weather(location: str) -> str:
    """날씨 리뷰 프롬프트 템플릿"""
    return f"{location}의 이번 주 날씨를 분석하고 외출 시 주의사항을 알려주세요."

Claude Desktop에 연결하기

서버가 완성되었다면 이제 Claude Desktop에 등록하여 실제로 사용할 수 있습니다.

설정 파일 수정

운영체제별 설정 파일 위치는 다음과 같습니다.

  • macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
  • Windows: %APPDATA%\Claude\claude_desktop_config.json

설정 파일에 다음 내용을 추가합니다.

{
  "mcpServers": {
    "weather": {
      "command": "uv",
      "args": [
        "--directory",
        "/절대/경로/my-mcp-server",
        "run",
        "server.py"
      ]
    }
  }
}

설정을 저장한 후 Claude Desktop을 재시작하면 채팅 입력창 근처에 망치(🔨) 아이콘이 표시됩니다. 이를 클릭하면 등록된 도구 목록을 확인할 수 있습니다.

디버깅과 테스트

MCP 서버 개발 중에는 MCP Inspector라는 공식 디버깅 도구를 활용하면 매우 편리합니다.

npx @modelcontextprotocol/inspector uv run server.py

브라우저에서 도구를 호출해보고 응답을 확인할 수 있어, Claude Desktop에 연결하기 전에 동작을 검증하는 데 유용합니다.

흔히 발생하는 문제

실전에서 자주 마주치는 이슈는 다음과 같습니다.

  • 절대 경로 누락: 설정 파일에는 반드시 절대 경로를 사용해야 합니다.
  • 로그 출력 위치: stdio 트랜스포트를 사용할 때는 print()를 stdout에 쓰면 안 됩니다. 로깅은 stderr로 보내야 합니다.
  • 비동기 처리: 외부 API 호출은 반드시 async/await를 사용해야 응답성이 유지됩니다.

보안 고려사항

MCP 서버는 AI 모델에게 시스템 권한을 일부 위임하는 것이므로 보안에 유의해야 합니다.

  1. 입력 검증: 사용자 입력을 그대로 셸 명령에 전달하지 마세요.
  2. 권한 최소화: 파일 시스템 접근은 필요한 디렉터리로 제한하세요.
  3. 민감 정보: API 키는 환경 변수로 관리하고 코드에 하드코딩하지 마세요.
  4. 로컬 우선: 외부에 공개되는 서버는 인증 메커니즘을 반드시 추가하세요.

다음 단계: 프로덕션 수준의 서버 만들기

기본 서버를 만들었다면 이제 더 실용적인 기능들을 추가해 볼 수 있습니다.

  • 데이터베이스 연동: SQLite, PostgreSQL과 연결하여 실제 데이터 조회
  • 외부 SaaS API: GitHub, Notion, Slack 등의 API 래핑
  • HTTP 트랜스포트: stdio 대신 SSE나 streamable HTTP를 사용해 원격 서버 구축
  • 상태 관리: 세션 기반 상태 유지 및 캐싱

Anthropic의 공식 GitHub(modelcontextprotocol/servers)에는 GitHub, Google Drive, Slack, PostgreSQL 등 다양한 레퍼런스 서버 구현이 공개되어 있어 학습에 큰 도움이 됩니다.

마치며

MCP는 단순한 API 래퍼가 아니라, AI 에이전트 생태계의 상호운용성을 책임지는 인프라입니다. 한 번 만든 MCP 서버는 Claude뿐 아니라 MCP를 채택하는 모든 호스트 애플리케이션에서 작동합니다.

처음에는 공식 SDK가 제공하는 FastMCP 클래스로 가볍게 시작하고, 필요에 따라 점진적으로 기능을 확장하는 것이 좋습니다. AI 에이전트 시대에 직접 도구를 만들어보는 경험은 분명 큰 자산이 될 것입니다.
